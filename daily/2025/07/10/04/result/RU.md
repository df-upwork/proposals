Правильный способ решения вашей задачи — использовать официальный для WP Engine инструмент https://github.com/wpengine/github-action-wpe-site-deploy
1) Вместо ручного копирования среды изменения в коде и плагинах фиксируются в системе контроля версий Git и отправляются в репозиторий (например, на GitHub). 
2) Этот push автоматически запускает процесс развёртывания на среду production с помощью GitHub Actions.
3) `github-action-wpe-site-deploy` предоставляет параметр `SCRIPT`, который позволяет указать путь к bash-скрипту. 
Этот скрипт будет выполнен на целевом сервере WP Engine сразу после успешного завершения развёртывания файлов.
Внутри этого скрипта можно использовать интерфейс командной строки WP-CLI, предустановленный на всех средах WP Engine, для вызова необходимой PHP-функции в кастомном плагине.
4) Этот правильный способ соответствует современным практикам DevOps. 
Он обеспечивает полный аудиторский след через историю коммитов Git, высокую надёжность и предсказуемость. 
Процесс становится повторяемым и автоматизированным, что снижает риск человеческой ошибки. 
Данный подход использует официально поддерживаемые и документированные функции платформы WP Engine.
