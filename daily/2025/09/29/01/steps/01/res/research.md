#
Система отчетности в D365 BC поддерживает несколько типов макетов, которые определяют внешний вид и структуру генерируемых документов. 
Основными из них являются Word, RDLC и Excel. 
Понимание различий между этими типами имеет решающее значение, поскольку наиболее жизнеспособное нативное решение проблемы применимо исключительно к одному из них.


# Макеты RDLC (Report Definition Language Client)
Это мощный, но сложный формат, основанный на технологиях SQL Server Reporting Services (SSRS). 
Разработка и модификация RDLC-макетов требует специальных инструментов, таких как Microsoft Report Builder или Visual Studio Report Designer, и навыков разработчика. 
Эти макеты обеспечивают «пиксельную точность» (pixel-perfect) и высокую степень контроля над сложными структурами данных, но они менее гибки для конечных пользователей и не подходят для простого решения проблемы со шрифтами в облаке.

# Макеты Word
Этот тип макетов основан на стандартных документах Microsoft Word (.docx). 
Их главное преимущество — простота и доступность кастомизации для пользователей, не обладающих техническими навыками. 
Процесс модификации макета Word включает в себя экспорт файла из D365 BC, его редактирование в приложении Microsoft Word с использованием стандартных функций форматирования и панели «Сопоставление XML» (XML Mapping Pane) для привязки полей данных, и последующий импорт измененного файла обратно в систему. 

Именно эта возможность офлайн-редактирования файла .docx является ключом к потенциальному решению проблемы. 
Она позволяет манипулировать содержимым файла макета на уровне, недоступном для RDLC-макетов из интерфейса BC, и открывает возможность для встраивания данных шрифта непосредственно в сам файл макета.
