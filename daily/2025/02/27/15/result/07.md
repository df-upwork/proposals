1) To filter leads on Salesforce export, I recommend a «Formula Field», e.g.: `IF(BEGINS(<account number> , "DLR-" ), "Dealer", "Customer")`.
In the standard Salesforce configuration, the `AccountNumber` field belongs to the `Account` object rather than the `Lead` object: in other words, it is not stored at the Lead stage and only appears after  Leads have been converted to Accounts.
To apply the filter at the Lead stage, create and populate a custom field on the `Lead` object and use it in the formula above.
2) Business Central (`BC`) and Dynamics 365 Sales (`S`) have a built-in synchronization (via «Dataverse Connection» and «Integration Table Mappings» in `BC`), but it does not fulfill the requirements of points 2 and 3 of your description, so I recommend Microsoft Power Automate (`PA`) to implement these requirements.
3) The inventory synchronization from `BC` to `S` with `PA`:
3.1) In `BC`, the inventory data are stored in the `Items` table.
3.2) The current stock quantity (`Inventory`) can be obtained from the `Item Ledger Entries` table or aggregated data in the `Items` table.
3.3) Open «Web Services» in `BC` and set up an endpoint, e.g.: `https://api.businesscentral.dynamics.com/v2.0/{tenant-id}/{environment}/ODataV4/Company('YourCompany')/Items`.
3.4) In `S` (without additional modules like Field Service or Supply Chain), there are no inventory fields by default.
So, create these fields: e.g., `Available Inventory`.
3.5) In `PA`, set up connections to the both systems:
3.5.1) `BC`: use the `Microsoft Dynamics 365 Business Central` connector.
3.5.2) `S`: use the `Microsoft Dataverse` connector.
3.6) Create a flow in `PA`:
3.6.1) Set up a trigger: «Automated cloud flow» or «Scheduled cloud flow».
3.6.2) Add the `Get rows` action from the `Microsoft Dynamics 365 Business Central` connector and specify the `Items` table.
3.6.3) Add the `List records` action from the `Microsoft Dataverse` connector and specify the `Products` entity in `S`.
3.6.4) Add the `Update a record` action from the `Microsoft Dataverse` connector:
- Specify the `Products` entity.
- Match the record from `S` with the data from `BC` by `Product Number` (corresponding to `Item No.`).
- Update the `Available Inventory` field with the value from the `Inventory` field of the `Items` table.
3.6.5) For new products, add the `Create a new record` action:
- Specify the `Products` entity.
- Populate the required fields and also `Available Inventory`.
3.6.6) Create a «Price List Item» record to associate the product with a price list.
Without this, it will not be possible to correctly add the product to an opportunity or quote.
Use `PA` to synchronize prices with `BC`.
3.7) Set up the mapping between `BC` and `S` fields in the flow actions.
In the `Update a record` or `Create a new record` action, specify:
- `Product Number` = `Item No.` from `BC`.
- `Name` = `Description` from `BC` (or another field containing the product name).
- `Available Inventory` = `Inventory` from `BC`.
4) The transfer of «Past Due» / «Credit Hold» / «YTD amount» is done similarly to point 3.
5) The transfer of dealers and customers from Salesforce to `BC` and `S` can be done via `PA` (similarly to point 3) or via CSV. 
`S` supports CSV import.
`BC` supports Excel import via «RapidStart Services», so CSV must be converted to Excel.
In `BC`, dealers and customers can be stored in the `Customer` table.
In `S`, dealers and customers can be stored in the «Account» entity.
6) To map fields in `S` according to Salesforce, use «Settings» → «Advanced Settings» → «Customizations» → «Customize the System» in `S`.
7) To transfer email templates from Salesforce to `S`:
7.1) Make a list of all merge fields used in Salesforce templates.
7.2) Map these fields in `S` (point 6 above).
7.3) Configure «Email Templates» in `S` («Settings» → «Templates» → «Email Templates»).
7.4) When creating or editing a template, insert the mapped fields.
7.5) If an automatic email sending by a trigger is required (analogous to «Workflow rules» and «Process Builder» in Salesforce), use the `S` and `PA` tools:
7.5.1) Open `PA` → «Create» → «Automated cloud flow» (or «Scheduled cloud flow»).
7.5.2) Add a trigger (e.g., «When a record is created» on the «Lead» entity).
7.5.3) In the email sending step, select the created «Email Template» from `S` and insert the required fields from Dataverse.
7.6) If the email sending scenario is very complex, use the «Send an email (V2)» action in `PA`.
7.7) To generate a PDF attachment or more complex documents, use «Document Templates» / «Word Templates»: it allows to gather data from entities (Account, Opportunity, etc.) and automatically insert them into a Word document.
Then, with `PA`, it is possible to generate a PDF file from this Word template and attach it to an email or save it in the CRM record.
