Ниже представлен анализ способов решения задачи (`T⁎`).

## Mᚖ₁: Синхронизация через сервер (Exchange / Microsoft 365)

### Суть
Если клиент использует учетную запись Microsoft Exchange, Microsoft 365 или Outlook.com, все данные (контакты, пользовательские поля, группы) хранятся на сервере.
Также на сервере хранится Мастер-список категорий (Master Category List, MCL), который определяет названия и цвета категорий (Источник 3.2).
Для миграции достаточно настроить ту же учетную запись в Classic Outlook на новом компьютере, и данные синхронизируются автоматически.

### Оценка (§2.3)
99/100

### Достоинства (§2.2.2)
Это самый простой и прозрачный метод для пользователя.
Вмешательство пользователя минимально и ограничивается настройкой учетной записи.
Гарантируется полная и точная сохранность всех данных, включая пользовательские поля, группы и цвета категорий.

### Недостатки (§2.2.1)
Метод применим только для учетных записей Exchange, Microsoft 365 или Outlook.com.
Он неприменим для учетных записей POP.
Он неприменим для учетных записей IMAP, так как протокол IMAP не поддерживает синхронизацию контактов и категорий (Источник 5.2).

## Mᚖ₂: Прямое копирование основного файла PST (для POP)

### Суть
Если клиент использует учетную запись POP, все данные хранятся локально в файле данных Outlook (PST) (Источник 5.1).
Метод заключается в копировании этого файла PST целиком со старого компьютера на новый при закрытом приложении Outlook.
На новом компьютере Outlook настраивается на использование этого скопированного файла в качестве основного хранилища данных.

### Оценка (§2.3)
95/100

### Достоинства (§2.2.2)
Это самый надежный метод для локальных данных, так как копируется вся база данных целиком.
Гарантируется сохранение пользовательских полей и групп контактов.
MCL и цвета также сохраняются, так как они хранятся в основном файле PST (Источник 3.2).

### Недостатки (§2.2.1)
Метод применим, только если данные хранятся в файле PST (учетные записи POP).
Он неприменим для файлов OST (используемых IMAP/Exchange).
Переносятся все данные из файла (почта, календарь), а не только контакты.

## Mᚖ₃: Копирование на уровне папки (Folder-Level Copy)

### Суть
Этот метод предполагает получение доступа к исходным данным контактов в профиле Outlook на новом компьютере и использование функции «Copy Folder» (Копировать папку) для переноса папки «Контакты» в новое хранилище.
Исходный файл данных (PST) подключается к новому профилю Outlook как дополнительное хранилище («Файл» > «Открыть и экспортировать» > «Открыть файл данных Outlook»).
Если контакты находятся в файле OST (например, локальные папки при IMAP), этот файл OST необходимо сначала конвертировать в PST с помощью Mᚖ₄ или Mᚖ₅.

### Оценка (§2.3)
90/100

### Достоинства (§2.2.2)
Этот метод обеспечивает высокую точность копирования данных, включая пользовательские поля и группы контактов.
Он позволяет интегрировать старые контакты в новый профиль, не заменяя его целиком.
Метод обходит ненадежный Мастер импорта/экспорта, выполняя прямое копирование объектов базы данных.

### Недостатки (§2.2.1)
Метод требует от пользователя аккуратного управления несколькими файлами данных в интерфейсе Outlook, что может быть сложно для `ꆜ`.
В сценарии IMAP (данные в OST) этот метод зависит от предварительной конвертации OST в PST (Mᚖ₄ или Mᚖ₅).
MCL (цвета) не переносится автоматически и потребует применения Mᚖ₇.

## Mᚖ₄: Использование стороннего программного обеспечения

### Суть
Использование специализированных инструментов для миграции данных Outlook.
Это включает конвертеры OST в PST, которые необходимы для надежного извлечения локальных данных (контактов) из OST-файлов, если используется IMAP (Источник 8.1, 8.3).
Также существуют инструменты для прямого переноса данных между профилями или для точного экспорта/импорта MCL с сохранением цветов.

### Оценка (§2.3)
85/100

### Достоинства (§2.2.2)
Эти инструменты разработаны для сохранения пользовательских полей, обходя ограничения встроенного экспорта (Источник 2.2).
Они часто являются единственным надежным решением для извлечения данных из OST-файлов в сценарии IMAP.
Они предоставляют надежные механизмы для переноса MCL с точным сохранением цветов.

### Недостатки (§2.2.1)
Качественное программное обеспечение обычно является платным.
Необходимо проверить надежность и безопасность выбранного инструмента.
Требуется установка и настройка дополнительного программного обеспечения.

## Mᚖ₅: Использование мастера импорта/экспорта PST

### Суть
Использование встроенной функции Outlook: «Файл» > «Открыть и экспортировать» > «Импорт и экспорт» > «Экспорт в файл» > «Файл данных Outlook (.pst)», выбирая только папку «Контакты».
Полученный файл импортируется на новом компьютере с помощью того же мастера.
Этот метод также используется для экспорта данных из OST в PST, если профиль доступен (Источник 8.1).

### Оценка (§2.3)
40/100

### Достоинства (§2.2.2)
Это встроенная функция Outlook, которая интуитивно понятна.
Метод позволяет выборочно экспортировать только папку «Контакты».

### Недостатки (§2.2.1)
Существует значительный риск потери пользовательских полей при использовании этого мастера (Источник 2.1, 2.2).
Мнение `ꆜ`, что этот метод не работает для пользовательских полей, подтверждается экспертами.
MCL (цвета) не экспортируется этим методом.

## Mᚖ₆: Ручные методы (CSV или Копирование/Вставка)

### Суть
Экспорт контактов через формат CSV с ручным сопоставлением полей.
Альтернативно, настройка табличного представления (List View) в Outlook с добавлением всех пользовательских полей и последующим копированием (Ctrl+C) и вставкой (Ctrl+V) в Excel для последующего импорта.

### Оценка (§2.3)
10/100

### Достоинства (§2.2.2)
Используются только стандартные средства (Outlook, Excel).
Метод копирования/вставки может позволить извлечь пользовательские поля, если они отображены в представлении (Источник 2.1).

### Недостатки (§2.2.1)
Группы контактов (mailing lists) будут полностью потеряны.
Категории и их цвета будут потеряны.
Стандартный экспорт в CSV часто игнорирует пользовательские поля (Источник 2.2).
Процесс настройки представлений и маппинга полей чрезвычайно сложен и трудоемок для нетехнического пользователя.

## Mᚖ₇: Перенос списка основных категорий (MCL)

### Суть
Это вспомогательный метод для восстановления цветов категорий, если они были потеряны (например, при использовании Mᚖ₃ или Mᚖ₅).
Вариант 1: Использование функции «Upgrade to Color Categories» (Обновить до цветных категорий) (Правая кнопка мыши на корневом элементе хранилища > Свойства); Outlook восстановит названия категорий, но цвета могут быть назначены случайным образом (Источник 3.2).
Вариант 2: Использование специализированных инструментов (часть Mᚖ₄) или скриптов для точного экспорта/импорта MCL.

### Оценка (§2.3)
Н/П (Вспомогательный метод)

### Достоинства (§2.2.2)
Метод решает конкретную проблему потери цветовых кодов.
Восстанавливает визуальное представление категорий.

### Недостатки (§2.2.1)
Встроенная функция «Upgrade to Color Categories» не гарантирует сохранение исходных цветов.
Точный перенос MCL требует использования специализированных инструментов, что усложняет процесс.

# Вердикт (§2.4)

Анализ подтверждает опасения `ꆜ`.
Стандартный Мастер импорта/экспорта (Mᚖ₅), даже при выборе формата PST, ненадежен для переноса пользовательских полей (Источник 2.1, 2.2).
Методы на основе CSV (Mᚖ₆) не подходят, так как теряют группы контактов и цвета.

Для гарантированного сохранения всех данных (пользовательских полей, групп и цветов категорий) необходимо использовать методы, которые переносят структуру базы данных напрямую.
Оптимальный метод критически зависит от типа учетной записи электронной почты клиента.

**Сценарий 1: Учетная запись Exchange / Microsoft 365**
Данные хранятся на сервере.
▶ Рекомендуется **Mᚖ₁ (Синхронизация через сервер)**.
Миграция произойдет автоматически после настройки учетной записи на новом компьютере.

**Сценарий 2: Учетная запись POP**
Данные хранятся в локальном файле PST.
▶ Рекомендуется **Mᚖ₂ (Прямое копирование основного файла PST)**.
Это гарантирует полный перенос всех данных и настроек, включая точные цвета категорий.

**Сценарий 3: Учетная запись IMAP или Интеграция в существующий профиль**
Контакты хранятся локально (часто в файле OST при IMAP) или в отдельном PST.
Файлы OST нельзя переносить напрямую (Источник 8.3); данные необходимо конвертировать в PST.
Поскольку встроенный экспорт (Mᚖ₅) ненадежен, а сохранение точных цветов является требованием, рекомендуется комплексный подход:
▶ Использовать **Mᚖ₄ (Стороннее ПО)**: конвертер OST в PST для надежного извлечения контактов с пользовательскими полями и группами, а также инструмент для экспорта/импорта MCL для сохранения цветов.
▶ Затем использовать **Mᚖ₃ (Копирование на уровне папки)** для переноса контактов из полученного PST в новый профиль.
