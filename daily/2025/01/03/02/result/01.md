Когда разработка и тестирование долгое время ведётся на Windows без постоянной проверки результата на Linux (или наборот), то подобные проблемы возникают сплошь и рядом.
По моему 25-летнему опыту, вот мой рейтинг самых популярных причин:
1) Лидер рейтинга, конечно — это слеши: `\` и `/`.
2) Серебряный призёр рейтинга — это регистр букв.
Очень просто где-нибудь пропустить, например `I` вместо `i`: для Windows разницы нет, а в Linux всё упадёт.
3) Бронзовый призёр рейтинга — это библиотеки-зависимости.
Возможно, для ваш проблемный экран использует библиотеку, которая у вас на Windows уже установлена, а на голом Linux, который вы используете для тестирования — не уставлена.
4) Кроме тройки призёров рейтинга есть и чуть менее популярные причины, типичные именно для Electron: например, отличия в environment variables.
Что забавно, специфические для Electron причины, тем не менее, по популярности значительно уступают универсальным лидерам моего рейтинга.
