| Пункт | Вероятность (%) | Пояснение |
|-------|----------------|-----------|
| 5 | 85% | Неправильно выстроенные связи между таблицами — фундаментальная проблема Power BI. Если таблицы некорректно связаны, новые данные не будут правильно отображаться в визуализациях. Это часто первое, что нужно проверить. |
| 7 | 80% | Ключевые поля критически важны для связывания данных между таблицами. Если они неправильно настроены (выбраны не те столбцы, значения не уникальны и т.д.), новые записи не будут правильно связываться с измерениями и не появятся в визуалах. |
| 8 | 75% | Несоответствие типов полей для связей (например, текст в MySQL, но число в Power BI) — распространенная проблема, которая мешает правильному соединению новых данных. Особенно вероятно, если структура MySQL изменилась. |
| 10 | 70% | Хотя клиент отметил, что дело "не в простой проблеме с фильтрами", более сложная фильтрация через отношения все равно может быть причиной. Особенно двунаправленная фильтрация может создавать неожиданные контексты фильтров, скрывающие новые данные. |
| 36 | 65% | M-скрипты Power Query часто содержат жёстко заданные списки полей или трансформации. Если структура MySQL изменилась, а скрипты не обновлены, новые поля данных не будут загружены. |
| 37 | 65% | Если шаги трансформации в Power Query опираются на столбцы, которые были переименованы или удалены, вся цепочка трансформаций может сломаться, препятствуя загрузке новых данных. |
| 14 | 60% | В режиме Import данные обновляются только по расписанию или при ручном обновлении. Клиент может не понимать эти ограничения и ожидать обновления в реальном времени. |
| 15 | 60% | Сбои запланированного обновления часто встречаются в Power BI Service. Если обновление не выполняется, а клиент не проверяет историю обновлений, он может думать, что данные не обновляются. |
| 6 | 55% | Проблемы с типами данных между MySQL и Power BI (даты, числа или текст) очень распространены и могут вызывать ошибки преобразования данных при обновлении. |
| 12 | 55% | Для локальных данных проблемы со шлюзом (Gateway) встречаются очень часто. Если шлюз неправильно настроен или не работает, Power BI Service не может получить доступ к базе данных. |
| 13 | 50% | Проблемы с учетными данными после изменения паролей или учетных записей могут помешать аутентификации Power BI в MySQL, вызывая сбои обновления. |
| 4 | 50% | Изменения схемы базы данных без соответствующих обновлений в модели Power BI — распространенная проблема, ведущая к проблемам с обновлением. |
| 16 | 45% | При использовании режима DirectQuery проблемы с разрешениями или конфигурацией сервера могут препятствовать выполнению запросов к новым данным. |
| 17 | 45% | Хотя базовые фильтры могли быть проверены, сложные комбинации фильтров или фильтры на уровне отчета могут по-прежнему скрывать новые данные. |
| 18 | 40% | Если настроена безопасность на уровне строк (RLS), новые данные могут отфильтровываться на основе правил безопасности, о которых клиент может не знать. |
| 19 | 40% | Если изменения внесены в Power BI Desktop для адаптации к новым данным, но не опубликованы в службе, онлайн-дашборды не будут показывать обновления. |
| 35 | 40% | Если связи разрушены из-за изменений схемы или ключей, это будет препятствовать правильной визуализации новых записей. |
| 9 | 40% | Если существует несколько связей, но та, которая нужна для новых данных, неактивна, это может препятствовать правильной визуализации. |
| 43 | 40% | Если поток данных в MySQL изменился для использования новых таблиц, но Power BI по-прежнему указывает на старые, новые данные будут невидимы в отчетах. |
| 33 | 35% | Переименование или удаление полей в MySQL без обновления Power BI вызовет сбои запросов или отсутствие данных. |
| 34 | 35% | Неожиданные изменения типа столбца в MySQL без соответствующих обновлений в Power BI могут вызвать ошибки преобразования при обновлении. |
| 38 | 35% | Проблемы с конфигурацией запланированного обновления могут препятствовать автоматическим обновлениям данных в Power BI Service. |
| 32 | 30% | Новые поля, добавленные в MySQL, не появятся в Power BI, если модель не обновлена для их включения. |
| 42 | 30% | Если Power BI работает с устаревшими метаданными схемы и не обновляет их, он может не видеть изменения схемы или новые поля. |
| 39 | 30% | Изменения в структуре ключей базы данных без обновления связей Power BI будут препятствовать правильному соединению данных. |
| 11 | 25% | Условные столбцы в Power Query, которые фильтруют на основе шаблонов, могут исключать новые данные, если эти шаблоны изменились. |
| 20 | 25% | Новые записи с пустыми значениями в ключевых полях могут быть исключены из отношений или отфильтрованы настройками по умолчанию. |
| 40 | 25% | Реструктуризация базы данных (разделение таблиц) без соответствующих обновлений модели Power BI приведет к несогласованности данных. |
| 22 | 20% | Блокировки транзакций обычно вызывают временные проблемы, а не постоянное "неотображение новых данных". |
| 21 | 20% | Проблемы с индексацией менее вероятно полностью предотвратят отображение данных (vs вызовут проблемы с производительностью). |
| 41 | 20% | Повторное использование имен таблиц/столбцов с разными структурами — это конкретный сценарий, который встречается реже. |
| 23 | 15% | Проблемы с временными таблицами обычно влияют на весь набор данных, а не только на новые данные, и менее распространены в продакшен-среде. |
| 24 | 15% | Проблемы со слоями промежуточного хранения требуют специфических процессов ETL, которые могут быть не актуальны для всех сред. |
| 31 | 15% | Отсутствие первичных ключей может вызвать проблемы с отношениями, но меньше непосредственно препятствует полному отображению данных. |
| 25 | 15% | Пересекается с другими проблемами ключевых полей (#7), но менее вероятна как самостоятельная причина. |
| 26 | 15% | Проблемы с кардинальностью могут вызвать проблемы с отношениями, но обычно обнаруживаются на этапе начального моделирования. |
| 27 | 15% | Конкретный случай фильтрации по устаревшей схеме, пересекающийся с другими проблемами фильтрации. |
| 28 | 10% | Проблемы с инкрементальным обновлением характерны для конфигураций, использующих эту функцию, которая не является настройкой по умолчанию. |
| 29 | 10% | Пересекается с несколькими другими причинами, связанными со структурой, и менее специфична. |
