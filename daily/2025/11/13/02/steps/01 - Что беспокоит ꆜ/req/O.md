## 2.3. Description
`PD` ≔ 
```text
#
Demė Health Intelligence (DHI) is developing a patent-pending framework to securely medical data for academic research.

We’re working with an Ivy League US institution to run a pilot that tokenizes and de-identifies patient data in their secure environment.

We need a skilled Python Data Engineer to create a simple, auditable prototype that can run fully offline and produce a de-identified dataset linking the two systems.

Seeking a healthcare data engineer to develop a secure, HIPAA-compliant tokenization and data integration module that links medical EHR systems. 
The module will ingest data via FHIR/HL7 APIs, tokenize limited identifiers (DOB, ZIP, sex) using SHA-256 or equivalent hashing, and map data to OMOP and FHIR standards for downstream analytics.

# Responsibilities:
- Build Python-based tokenization pipeline (non-reversible identifiers).
- Develop FHIR/HL7 ingestion layer for EHR systems
- Map data to OMOP for normalized schema.
- Validate linkage accuracy using synthetic test data.
- Deploy inside HIPAA-compliant environment.

# Skills: 
Python, HL7, FHIR, OMOP, Datavant/HealthVerity familiarity, AWS/GCP healthcare stack.

# Deliverable
```
