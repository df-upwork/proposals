
1. Пункт 2.2.4: Наблюдается терминологическая неточность. В экосистеме Next.js (App Router) термин "Request Memoization" преимущественно используется для обозначения автоматической мемоизации (дедупликации) запросов, выполненных через `fetch` API, в рамках одного цикла рендеринга. Для мемоизации данных, полученных иными способами (например, через ORM, GraphQL клиенты или прямые запросы к БД), используется функция React `cache`. В пункте 2.2.4 под заголовком "Игнорирование Request Memoization" приводится пример отказа от использования React `cache`. Хотя суть описанной проблемы (дублирование идентичных запросов из-за отсутствия ручной мемоизации) верна, использование термина "Request Memoization" в этом контексте не соответствует его определению в документации Next.js и может вызвать путаницу.
