Пункт: II.1.3
Замечание: Неточный акцент в описании структуры затрат CloudWatch (потенциально вводящий в заблуждение). Утверждается, что включение высокоточных метрик (high-resolution custom metrics) «increases API usage costs due to more frequent data transmission» (увеличивает затраты на использование API из-за более частой передачи данных). Технически это верно: более частая передача данных увеличивает количество API-вызовов (PutMetricData). Однако основным фактором затрат в CloudWatch, как правило, является стоимость хранения метрик (cost per metric per month), а не стоимость API-вызовов. Акцентирование внимания только на затратах API может создать неполное представление о реальной структуре расходов при использовании высокоточных метрик.
Степень уверенности: 85


3.
Пункт: II.2.3
Замечание: Логическая неточность (неполное сравнение) в описании интеграционных возможностей. Утверждается: «The alerting capabilities offer fewer integrations with the broader AWS ecosystem [...] compared to CloudWatch Alarms» (Возможности оповещения предлагают меньше интеграций с более широкой экосистемой AWS [...] по сравнению с CloudWatch Alarms). Это утверждение верно только в отношении прямых (native) интеграций, встроенных в консоль AWS. Однако оно игнорирует наличие в Netdata полноценной поддержки веб-хуков (webhooks). Веб-хуки позволяют реализовать интеграцию с ключевыми сервисами AWS (SNS, Lambda через API Gateway, EventBridge) и достигать аналогичных функциональных результатов (например, автоматического реагирования). Функциональные возможности интеграции сопоставимы, и утверждение о «меньшем количестве интеграций» без этой оговорки является неточным и вводит в заблуждение.
Степень уверенности: 90
