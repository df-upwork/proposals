Для реализации ваших макетов в Grafana потребуются следущие доработки:
1) Специальные «выезжающие» правые панели (макеты `MP002-S`, `MP003-S`, `MP005-L`, `MP006-L`, `MP007-L`).
Стандартно Grafana не открывает подобную «панель деталей» по клику в таблице. 
Есть лишь popup для отдельных полей или переход на другой дашборд.  
Для реализации такого «сайдбара», как в макетах нужно программировать плагин (React/TypeScript)  
2) Специальные типы полей в таблицах (иконки батареи, «сотовый» сигнал, цветные бейджи + «+200%» и т. д.).
Для реализации этой функцональности тоже нужно программировать плагин (React/TypeScript).
3) Система «Score» (сумма метрик, проценты, тенденции).
Есть 2 пути реализации этой системы:
3.1) Рассчитывать «Score» в самом источнике данных.  
Если у вас есть база или сервис, который уже агрегирует (например, «(Температура × 0.3) + (Давление × 0.5) + (Вибрация × 0.2) = Score»), то можно записывать итоговые значения Score в БД (InfluxDB, Elasticsearch, Prometheus и т.д.).  
Тогда Grafana получит уже готовый показатель Score.  
Процент изменения (плюс/минус) можно тоже заложить в запрос (например, сравнивать текущее среднее за сутки с предыдущим периодом).
3.2) Делать агрегирование на стороне Grafana (Transformations)  
Если данные приходят в виде нескольких метрик (Temperature, Pressure, Vibration), то можно:  
3.2.1) Объединить их в одну табличку (join / merge) через Transformations.  
3.2.2) Добавить «Calculated field» (новая возможность Transformations в последних версиях Grafana), где прописать формулу вроде:  
 ```
 score = temperature * 0.3 + pressure * 0.5 + vibration * 0.2
 ```
3.2.3) Полученное поле назвать «Score».  
3.2.4) Для «+200%» (динамика) придётся либо: 
3.2.4.1) делать «time shift» и ещё одно поле – «Score (previous period)», затем считать `(score_current - score_previous)/score_previous * 100%`
3.2.4.2) использовать «Reduce / Moving average» и затем сравнить прошлое и настоящее в одном наборе.  
4) Ассоциативные связи:
- «Active Component» → «Associated Lubricator» (макет `MP003-S`)
- «Lubrication» → «Associated Components» (макет `MP007-L`)
Для реализации таких связей тоже нужно программировать плагин (React/TypeScript). 
5) Кастомная шапка/меню (Dashboards, Manage Outages, Insights, Account).
Для реализации такой шапки нужно программировать плагин с собственным Layout (частичная замена интерфейса). 
6) Вкладки на упомянутых в пункте 1 правых панелях: например, «Analysis» на макетах `MP002-S`, `MP006-L`, с несколькими графиками + кастомным описанием.
Для реализации таких вкладок тоже нужно программировать плагин, в дополнение к плагину пункта 1.
7) Обработка нажатий кнопок «RAISE WORK ORDER» и «APPLY CHANGES» на макете `MP005-L`.
Для этого тоже нужно программировать плагин.
